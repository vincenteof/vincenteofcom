# Core Beliefs - 项目核心信念

**最后更新**：2026-04-11

本文档记录了这个项目最重要、最根本的信念与设计哲学。  
AI Agent 在进行任何规划、设计或代码修改时，必须优先参考本文件。

## 1. 项目本质与定位
- 这个网站是一个**个人主权内容发布系统**，核心是内容创作者的长期实践记录。
- 我们不是在建立一个“专业投资/认知网站”，而是在公开记录一个**长期学习者与实践者**的思考过程。
- 核心资产排序：**内容质量 × 转化率 × 留存** > 技术复杂度 > 视觉效果。

## 2. 内容与语气风格
- 整体语气：**轻微人文感 + 诚实 + 克制**。
- 强调“我是一个学习者、实践者、记录者”，而非权威、专业人士或投资导师。
- 避免：炫耀收益、强调资历、公司背景、确定性结论。
- 接受：观点会改变、判断会修正、写作只是思考的中途记录。
- Landing Page 和所有文案必须保持这种“认真做事的人”而非“建立权威的人”的感觉。

## 3. 技术哲学
- 技术只是工具，**必须服务于内容发布和会员变现闭环**。
- 优先选择**简单、可靠、可维护**的方案，而不是最先进或最复杂的方案。
- 严格遵守 server/client 边界：会员文章正文必须 server-only，绝不允许未授权用户在客户端拿到全文。
- 时间字段统一使用 INTEGER（Unix timestamp 秒），保持一致性。
- 部署必须保持在 Cloudflare Workers + D1，追求极简全栈架构。

## 4. 产品与用户体验原则
- MVP 目标是**快速上线并形成内容 → 订阅 → 支付 → 解锁**的最小闭环。
- Landing Page 是最高优先级，完成后即可发布 v0.1。
- 会员部分更像是**持续更新的研究笔记**，而非单纯的付费墙。
- 尊重用户：不骚扰、不过度营销，保持克制和真诚。

## 5. 代码与架构信念
- 代码应简洁、可读、易理解。
- 优先使用 features/ 垂直切片组织业务逻辑（posts、membership、landing 等）。
- 任何重复问题都应视为系统性问题，需要通过更新 Harness（规则、文档、结构）来永久修复，而不是临时 patch。
- 架构漂移是比 bug 更严重的问题。

## 6. 个人核心价值观
- 长期结构 > 短期收益
- 持续调整与理解 > 给出标准答案
- 主权与选择权 > 被动跟随
- 公开记录过程本身就是有价值的
