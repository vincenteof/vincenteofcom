# Service Intake Form Requirements (MVP v0.1)

**最后更新**：2026-04-13  
**状态**：Draft (待确认后开发)  
**适用阶段**：Landing 转化优化前置阶段

## 1. 背景与目标
当前 Landing 的 Software Consulting 与 Global Investing Advisory 已有曝光，但咨询路径不闭环。  
本需求用于定义一个低复杂度、可验证的线索收集机制。

核心目标：
1. 为咨询意向用户提供明确下一步。
2. 在不引入注册流程的前提下收集结构化线索。
3. 让创作者可以快速筛选与回复高匹配线索。

## 2. 方案决策（当前）
### 2.1 信息架构决策
- 暂不拆分为两个独立咨询落地页。
- 先使用一个统一入口：`Service Intake Form`。
- 表单内通过 `需求类型` 进行分流（软件开发 / 出海投资 / 暂不确定）。

### 2.2 流程复杂度决策
- 暂不引入注册流程（不要求账号创建）。
- 先验证线索质量与转化率，再评估是否升级为多步骤 funnel。

## 3. 用户路径（MVP）
1. 用户在 Offerings 点击咨询相关 CTA。
2. 进入统一咨询入口（站内表单页）。
3. 填写并提交需求。
4. 看到成功反馈：已收到需求 + 预计响应时间 + 可补充联系方式。

## 4. 字段定义（MVP 最小集）
### 必填字段
1. `name`：称呼/姓名（2-40 字符）
2. `serviceType`：需求类型
   - software_consulting
   - global_investing_advisory
   - not_sure_yet
3. `goalContext`：当前阶段与目标（30-500 字符）
4. `budgetRange`：预算区间（枚举，见下）
5. `email`：联系邮箱（基础格式校验）

### 可选字段
1. `timeline`：期望开始时间（如：立即 / 1 个月内 / 3 个月内）
2. `contactAlt`：其他联系方式（微信/Telegram）

### 预算区间枚举（建议）
- below_5k
- 5k_to_20k
- 20k_to_50k
- 50k_plus
- undecided

## 5. 文案与语气要求
- 语气保持诚实、克制，不使用压迫式营销词。
- 每个字段提示应帮助用户快速填写，不增加心理负担。
- 成功态必须给出明确承诺：例如「通常在 48 小时内回复」。

## 6. 非目标（当前阶段不做）
- 不做账号注册与登录绑定。
- 不做多步问卷与复杂评分模型。
- 不做 CRM 深度集成与自动化打标。

## 7. 验收标准（MVP）
1. 用户可在 2 分钟内完成填写并提交。
2. 每条线索包含可用于筛选的关键字段（需求类型 + 目标 + 预算 + 联系方式）。
3. 提交后用户清楚知道下一步（等待回复时长、补充方式）。

## 8. 迭代策略
### Phase 1（当前）
- 上线统一咨询入口，验证线索质量。

### Phase 2（线索样本达到 10-20 条）
- 根据真实数据评估是否拆分为两条独立落地页。
- 优化 CTA 与问题项，提升匹配率。

### Phase 3（线索稳定）
- 评估是否引入注册流程与更深层的自动化分发。

## 9. 待确认项
1. 入口形态：**已定为站内自建表单页**（不依赖第三方付费服务）
2. 预算区间字段是否保留为必填
3. 响应 SLA 文案是否固定为「48 小时内回复」
