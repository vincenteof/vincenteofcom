# Posts Visual Consistency Requirements (MVP v0.1)

**最后更新**：2026-04-13  
**状态**：Implemented（P0 / P1 已完成）

## 1. 背景
Landing Page 已完成重做，视觉语言已升级（OKLCH 配色、中文优先排版、克制交互）。  
`posts` 相关页面尚未进行系统性一致性检查，存在潜在视觉漂移与体验断层风险。

## 2. 目标
1. 识别 posts 页面与新 Landing 视觉系统之间的不一致项。
2. 只处理视觉与交互一致性，不扩展到内容信息架构调整。
3. 形成可执行修复清单，避免零散 patch。
4. 目标不是机械复刻 Landing，而是在保持阅读舒适性的前提下，共享同一视觉语言与交互气质。

## 3. 检查范围
- 页面：
  - `/posts`（文章目录）
  - `/posts/:slug`（文章详情）
- 代码范围：
  - `src/routes/posts/index.tsx`
  - `src/routes/posts/$slug.tsx`
  - `src/lib/ui.ts`

## 4. 检查维度
### 4.1 视觉系统一致性
1. 色彩 token 使用是否与 Landing 一致（尤其 accent、line、text-soft）。
2. 字体层级与中英混排是否保持同一品牌语气。
3. 间距节奏（section spacing、容器宽度）是否出现明显断层。
4. 按钮样式是否与 Landing 使用同一交互语言（边框、圆角、hover、focus-visible、active）。
5. 正文内链接、列表链接、返回按钮等不同类型链接的视觉表现是否和谐。

### 4.2 交互一致性
1. 关键链接/按钮是否具备完整状态：default / hover / focus-visible / active。
2. 焦点可见性与触控目标是否满足统一标准。
3. 动效节奏是否遵守 `prefers-reduced-motion` 策略。

### 4.3 路径与语义一致性
1. CTA 语义与跳转目标是否一致。
2. 是否存在失效锚点或过时路径。

## 5. 本轮明确不做
1. 不调整 posts 页面内容结构与信息架构。
2. 不修改文章正文内容本身。
3. 不进行会员流程与支付流程重构。

## 6. 审计发现（已在本轮处理）
1. 文章详情页会员 CTA 曾指向 `/#membership`，与当前 Landing 结构不匹配。
2. posts 页面按钮/链接状态体系曾与 Landing 最新规范存在细节漂移。
3. 文章正文中的链接样式此前与 Landing 的链接语言不够和谐。

## 7. 修复优先级定义
- **P0**：阻断路径或明显错误（失效链接、不可访问关键入口）
- **P1**：影响主要体验一致性（CTA 行为不一致、焦点状态缺失）
- **P2**：视觉节奏不统一但不阻断任务
- **P3**：纯微调项

## 8. 验收标准
1. 完成 `posts` 全链路一致性审计并产出问题清单（含优先级）。
2. 至少解决全部 P0 与 P1 问题后再视为一致性达标。
3. 审计结果与修复方案更新到执行计划文档，便于回归追踪。
4. 至少覆盖以下重点：按钮风格统一、正文链接样式统一、失效 CTA 修复。

## 9. 已确认原则
1. Posts 页面不要求机械复刻 Landing。
2. 只要视觉风格一致、交互气质统一、阅读体验舒服即可。

## 10. 本轮执行结果
1. P0：member preview 中失效的 `/#membership` 已移除，改为真实可用的 `订阅长信` 入口。
2. P1：`/posts` 与 `/posts/:slug` 已统一按钮、返回链接、focus-visible 和 active 状态。
3. P1：正文内链接已统一为更克制的颜色与下划线逻辑，并补齐 hover / focus-visible 状态。
4. P2：文章目录列表 hover 背景、标题反馈和会员 badge 已做细节修正。
